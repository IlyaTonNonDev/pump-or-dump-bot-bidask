# Настройка ключей для Pump/Dump Bot

## Быстрая настройка

Запустите интерактивный скрипт для настройки всех ключей:

```bash
node setup_keys.js
```

Скрипт поможет вам:
- ✅ Ввести или использовать существующие ключи
- ✅ Зашифровать seed фразу кошелька
- ✅ Сгенерировать ключи шифрования
- ✅ Настроить сумму покупки
- ✅ Сохранить все в `secretkeys.env`

## Необходимые ключи

### 1. TELEGRAM_BOT_TOKEN ✅ (уже настроен)
- Получить от [@BotFather](https://t.me/BotFather)
- Добавьте токен в `secretkeys.env`

### 2. TON_API_KEY ✅ (уже настроен)
- Получить от [TON Console](https://tonconsole.com)
- Добавьте ключ в `secretkeys.env`

### 3. ENCRYPTED_WALLET_SEED ❌ (нужно настроить)
- Зашифрованная seed фраза кошелька (12 или 24 слова)
- Используйте `node encrypt_seed.js` или `node setup_keys.js` для шифрования

### 4. ENCRYPTION_KEY ❌ (нужно сгенерировать)
- 32-байтный hex ключ для шифрования
- Будет сгенерирован автоматически при использовании скрипта

### 5. ENCRYPTION_IV ❌ (нужно сгенерировать)
- 16-байтный hex вектор инициализации
- Будет сгенерирован автоматически при использовании скрипта

### 6. BUY_AMOUNT_TON ⚠️ (опционально)
- Сумма покупки в TON (по умолчанию: 1)
- Можно настроить в скрипте или добавить вручную

## Ручная настройка

Если вы предпочитаете настроить вручную:

### Шаг 1: Зашифруйте seed фразу

```bash
node encrypt_seed.js "your seed phrase here"
```

Скопируйте полученные значения:
- `ENCRYPTED_WALLET_SEED`
- `ENCRYPTION_KEY`
- `ENCRYPTION_IV`

### Шаг 2: Добавьте в secretkeys.env

Откройте `secretkeys.env` и добавьте:

```env
ENCRYPTED_WALLET_SEED=ваша_зашифрованная_фраза
ENCRYPTION_KEY=ваш_ключ_шифрования
ENCRYPTION_IV=ваш_iv
BUY_AMOUNT_TON=1
```

## Проверка настроек

После настройки проверьте, что все ключи на месте:

```bash
# Проверка наличия всех ключей
grep -E "^(TELEGRAM_BOT_TOKEN|TON_API_KEY|ENCRYPTED_WALLET_SEED|ENCRYPTION_KEY|ENCRYPTION_IV|BUY_AMOUNT_TON)=" secretkeys.env
```

Должны быть все 6 ключей.

## Безопасность

⚠️ **ВАЖНО:**
- Никогда не коммитьте `secretkeys.env` в git
- Убедитесь, что файл добавлен в `.gitignore`
- Никогда не храните реальные ключи в файлах документации
- Храните резервную копию ключей шифрования в безопасном месте
- Если ключи шифрования потеряны, seed фразу нельзя будет восстановить
